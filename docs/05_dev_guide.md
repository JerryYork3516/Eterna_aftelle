## 0. 三条最高优先级(违反一条就回到 Studio 的老路)

1. **先跑最小闭环,再砸大 token。** 不许一上来做大而全。
2. **所有 AI 只围绕"架构红线 + 当前一个小任务"工作。** 一个任务、一个 AI 改、另一个审,不许群殴。
3. **先冻结 DR Runtime 契约,再做漂亮粒子。** 契约错了后面全返工;身体可以后置。

## 1. Stage 7 的成功定义(最小闭环优先)

Stage 7 不是"做完所有 7.x",而是**先打通这一条命脉链路**:

```
Studio 导出 .digital_resident
  → Aftelle 加载 + 校验
    → Runtime 对话(7.0/7.1 可用 mock;7.4 居民打磨前接真实 LLM)
      → SessionStore / HostStateStore(带版本号)
        → Metal 粒子表现
          → Trace 可见
            → 安全边界不破
```

**这条链路通了,Stage 7 MVP 就成立了。** 双居民、屏幕指导、行业居民属于 Stage 7 Extended Demo,不是 MVP 前提。

> 注:G0 已锁定真实 LLM 来源:Stage 7 MVP 可 mock;真实 Provider 只能走 RuntimeCore Provider 链路。真实 LLM 在 7.4 居民打磨前必须接(mock 无法验证人格/套话)。

## 2. 开工第一件事:Stage 7.0 Calibration(标定 + 验证工作流)

不要直接进完整 7.1。先做一个**极简验证**,目的是验证"工作流"而非"代码":

- 开一个空 Xcode 项目
- 屏幕上只放 **约 10 个粒子**
- 用一个**手工做的测试 DR**(fixture),走通:**加载 DR → 改一下粒子逻辑 → 看到变化**
- 全程记录:**花了多少额度、AI 读了多少文件、卡在哪**

**这一步的产出不是功能,是三个答案:**
1. 我的额度到底够不够(用真实数据,不靠猜)
2. 工具协作流程顺不顺
3. 粒子盲测机制好不好使(见第 6 节)

跑完这个,再用它外推整个 Stage 7,并决定要不要升档。

## 3. 防烧钱四铁律(对应 Studio 的四个漏洞)

| Studio 的漏洞 | Stage 7 的铁律 |
|---|---|
| 一次读全项目 | **文件读取白名单**:每个任务只授权 AI 读指定目录/文件(见下) |
| bug 换 agent 群殴 | **一个 bug,一个 AI**:先定位 → 圈范围 → 一个 AI 改 → 改不好停下想,不换人 |
| 没想清就动手、推翻重来 | **先想清楚再写**:架构/需求先在 GPT/Dify 盘清楚,代码 AI 不当草稿纸 |
| 加需求牵连一片 | **优先加新不改旧 + 对照红线** |

**文件读取白名单(关键):**
在 AGENTS.md 里写死一条:
> 执行任何涉及多个文件的搜索或修改前,先报告**预计读取哪些文件、大致范围**,经我确认后再执行。不许为"理解全局"私自抓取大量不相关文件。

这一条直接堵住最大的 token 黑洞,无论计费方式如何都省。

## 4. Git 安全降落伞(防改崩)

AI 跑批处理、跨文件重构最容易把项目改崩。规矩:

- **任何跨文件大改之前,强制先 commit 一次。** 改崩了能一键回滚。
- 提交信息带阶段号:`[7.1] ...`
- 主分支保持可运行;大改动走分支。
- **私有仓库;API Key / 密钥 / 真实 DR 文件永不进 Git。**

写进 AGENTS.md:"进行跨文件修改前,必须先执行一次本地 commit 作为回滚点。"

## 5. DR Runtime 契约优先(防全局返工)

Aftelle 的核心不是粒子,是 **DR 文件能被稳定加载、校验、运行、升级**。所以:

- **先把 DR 加载/校验/Runtime 这条链路做稳,再投入粒子打磨。**
- DR 只读 + 带 revision/schema_version(红线 4)。
- SessionStore / HostStateStore 带 schema_version(红线 2),为将来迁移留结构;居民长期记忆读写只经 RuntimeCore ExecutionEngine。
- **安全边界即使 Stage 7 不做商业系统也要提前埋:**
  - Provider secret 由 Apple Keychain 持有;Aftelle 只提交/更新 `key_ref` 和非密钥配置;Key / Base URL / Model 都不进 DR / Node / Slot / Trace / Git
  - Aftelle 只做契约级浅校验(版本/大小/安全 flag/必要字段/未知高危字段);完整 DR schema 校验由 RuntimeCore 执行
  - 崩溃日志、权限边界预留接口

> 修正认知:DR 契约错了,粒子做得再漂亮也要返工。**先稳契约,后做身体。**

## 6. 粒子盲测机制(解决"AI 看不见屏幕")⭐

这是你这种**几乎全粒子、没传统 UI** 的 App 的特有难题:AI 写得出 Metal 粒子代码,但**看不见屏幕上的粒子是否偏色、掉帧、坐标错位**。解法是**别让 AI 靠"看",让它靠"数据"验证**:

**① 纯数据终端日志(主要手段)**
让粒子系统把关键状态打成结构化日志,AI 读日志就能判断对错,不用看屏幕:
- 粒子数量、关键坐标范围(边界框,不是每个粒子)、主色值
- 当前帧率(FPS,聚合)
- Avatar State(当前情绪/状态)
- 状态切换是否触发了对应的粒子变化

**⚠️ 日志必须降频,严禁逐帧逐粒子输出(否则烧爆 token / 撑爆上下文):**
- **只在状态切换时打一条**(idle→thinking 等),或**每秒聚合输出一次**(平均 FPS + 边界框 + 粒子数 + 当前 state)。
- 60 帧 × 几十粒子的 per-frame 坐标日志是禁止的——10 秒就能撑爆 Claude Code 上下文、烧光额度、卡死。
- 调试某一帧细节时,单独触发"快照一帧",而不是持续逐帧打。

**② Xcode Previews 截图验证(辅助手段)**
Xcode 26.3 的 agent 能捕获 Previews 截图自己看效果——用它做"视觉对不对"的粗判。

**③ 帧率硬指标(可判定验收)**
- 目标分辨率下 FPS ≥ 指定值(如 30 可用 / 60 目标)
- 掉帧打进日志,AI 能看到

**原则:凡是"看起来对不对"的判断,都要先变成"日志里的数字对不对"。** 这样 AI 这个"瞎子"也能验证自己的视觉代码。

## 7. 测试体系(执行层最大的补漏)

之前架构防守严密,但缺测试。Stage 7 至少要有:

| 测试 | 验什么 | 优先级 |
|---|---|---|
| DR fixture | 正常/错误 DR 能否正确加载/优雅报错 | **必须** |
| 粒子 FPS test | 帧率达标、不掉帧 | **必须** |
| runtime step test | 一轮对话链路是否正常 | 高 |
| provider fallback test | LLM/TTS 失败时是否优雅降级 | 中 |
| schema migration test | Memory 版本变了能否迁移 | 中(Stage 8 重) |
| UI smoke test | 启动→加载→对话不崩 | 高 |

谁写测试:**让 AI 写,你验收。** 但 fixture 和"什么算通过"由你定。

## 8. 工具分工(做减法,但保留你的结构)

```
Claude Code → 主开发:brain/(RuntimeCore)、schema、架构复审、复杂报错(读仓库、改、测试、小步提交)
Codex       → 备用开发 + 独立审查:前端/platform-macos/、UI 稳定性
GPT         → 架构/产品/流程顾问:想方案、拆阶段、写提示词、做取舍(不抢仓库代码)
Cursor      → 局部审查/重构,不与 Claude 抢同一块代码
Dify        → 模拟会议/角色讨论,非工程主控台
```

**主控台是:Git + AGENTS.md + DEVLOG.md + 任务清单。** 不是 Dify。

**额度提醒:** 开工前用 Claude Code 的 `/status` **自己确认**订阅额度与计费方式(网上说法不一,只有你账户给真相)。无论怎么计费,**圈定文件范围就是省钱**。

## 10. 改 Bug / 加需求流程

**改 Bug:** 先定位(可问 Claude Code,它读得了仓库)→ 圈范围 → 一个 AI 改 → 改不好停手回讨论 → 修好记进 DEVLOG。**不准换 AI 群殴。**

**加需求:** GPT/Dify 想清楚 → 能否只加新代码不改旧 → 对照六红线 → Claude Code 实现 → 记 DEVLOG。大改动当一个小阶段立项。

## 12. 六条架构红线(随时对照)

1. 大脑(Runtime)不碰平台,只走 HostEnv
2. Store 带 schema_version
3. UI / Runtime / DR Loader 三层不黏连
4. DR 只读 + revision
5. 粒子逻辑 / 画法分层
6. 交互按"意图"抽象(非鼠标/触摸)

## 13. 认知修正(来自多 AI 复核)

- **"Apple 顺延几乎免费"→ 修正为"比跨 Android/Windows 省很多,但不免费"**:iOS/visionOS 仍要处理触控、权限、能耗、ARKit、窗口模型。
- **"打磨居民完全不用动画布"→ 修正为"填人设不用,但加真实记忆/人格版本/多模态会牵动画布"**。
- **"Claude Code 不按 token 计费"→ 修正为"以 /status 实测为准"**,别信任何单一说法。
