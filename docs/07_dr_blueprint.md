数字居民 13 层蓝图 v5（全填空版）

用途：作为数字居民在画布中的完整描述模板。
结构：一级标题使用"一、"，二级标题使用"1."，三级为字段，每个字段后留填空位"：________"。
说明：本蓝图用于设计、审核、编译，不直接执行。字段为"可填"而非"必填"——有内容就填，无内容留空。造居民时由设计者口述需求，再决定填哪些字段。

⚠️ 重要：本蓝图是**长期愿景模板,非契约、非 schema,不得用于 Stage 7 coding**。其中执行代码、外部服务、技能系统、自动任务、云端部署、支付/社交账号权限、现实空间设备、社交等，均为 Stage 8/9+。Stage 7 仅使用身份/人格/安全/记忆边界/lattice/voice/disclosure 等核心设计输入。填居民时由设计者与 AI 判断哪些进 Stage 7 DR、哪些仅作未来记录。
v5 变更：强化蓝本非契约警示；对齐 `aftelle_runtime_boundary.md`，保留 per-resident 表现映射、三种记忆边界与自我披露策略；内容语言保持中文。

边界说明：
- 情绪 → 视觉 / 语音 / 节奏的具体映射属于每个居民自己的数据,不得写进 Aftelle kernel。
- 多居民记忆必须区分 Private Memory / Shared Session Context / Public Transcript。
- 自我披露策略是显式设计参数;真人感和去 AI 味不得覆盖安全诚实边界。
